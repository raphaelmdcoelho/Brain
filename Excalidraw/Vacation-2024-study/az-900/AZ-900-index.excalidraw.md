Paas: Here we have same as IaaS and in addition we have

    network controls, applications and identity and directory infrastructure

Saas: Here only Information and data, devices and accounts and identities

So:

1. Information and data, devices (mobile and PCs) and accounts and identities
are ALWAYS responsibility for the customer

Microsoft. The only exception for this is On-Premises scenarios.

3. Your service model will determine responsibility for things like:

- Operating systems
- Network controls
- Applications
- Identity and infrastructure ^cB9MvWpU

Cloud Models ^hjTLTWDq

Definition: Cloud models define the deployment type of cloud resources.
The three main cloud models are: private, public, and hybrid.

* Private cloud:
    - It's the delivery of IT services over the internet
    - It's used by a single entity
    - It provides much greater control for the company and its IT department
    - Great cost
    - Fewer benefits of a public cloud deployment
    - Can be hosted from own on site datacenter (on-premise)
    - It may also be hosted in a dedicated datacenter offsite
    - Potentially even by a third party that has dedicated that 
        datacenter to your company.

* Public cloud:
    - It is built, controlled, and maintained by a third-party cloud provider
    - The general public availability is a key difference between public and private clouds

* Hybrid cloud: 
    - It is a computing environment that uses both public and private clouds in an
        inter-connected environment

* Multi-cloud:
    - It is composed by multiple public cloud providers ^t4Mx2YXM

Azure Solutions for cloud model

* Azure Arc: It is a set of technologies that helps manage
your cloud environment ^0F33RofH
